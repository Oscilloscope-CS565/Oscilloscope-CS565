# Rubric: Group Project Update – In-Class Presentation (100 Points Total)

## 1 Project Overview & Purpose(5 points)
Clear introduction of project name, topic, problem being solved, and high-level description. Demonstrates context and understanding of the project domain.

## 2 Literature Review / Topic Understanding (5 points)

Shows research, conceptual clarity, or technical background related to the problem space. Demonstrates progress in understanding the topic.

## 3 Architecture – 4+1 Views (Comprehensive) (30 points)

Provides high-quality architectural documentation with at least the following views:• Enumerated & labeled requirements• Use case diagrams• Activity diagrams• Class diagrams• Sequence diagrams• Deployment diagram• Package diagram

Optional but encouraged:• Object diagrams• State diagrams

## 4 Code Structure & Quality (10 points)

Follows best practices:• One class per file• Meaningful, namespaced class and file names (e.g., scpMyClass)• Each class has a clearly defined responsibility

## 5 Demo Execution – Terminal Control (10 points)

Demonstrates the ability to run the scope/function generator from terminal. Executes commands such as:• scope start• sampleTime=1ms• sampleFor=10s• stop

## 6 Demo – Data Interaction (10 points)

Real-time display of scrollable data. Shows that changes in input affect output and demonstrates scaling and offsetting features.

## 7 Code-to-UML Mapping (10 points)

Explains how the code structure aligns with architectural diagrams. Draws clear connections between design and implementation.


## 8 Debugging Demonstration (5 points)

Debugging Demonstration

Presenter demonstrates breakpoints and steps through the code. Explains the runtime behavior and logic flow during execution.


## 9 Presentation Quality (5 points)

Clear, organized, and professional presentation. Well-structured slides. All group members participate and stay within time.

## 10 Submission Completeness (10 points)

Required Submission:• PowerPoint slides (or PDF) • Zipped source code folder

Exact compile/run command provided (you will be graded based on whether the code compiles successfully on instructor’s machine)
