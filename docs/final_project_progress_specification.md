# Final Project Progress -- Group
Group Project Update in Front of Class:
A Power Point Presentation containing:

## 1. A presentation about the progress of your final project:  Project name, topic, description

## 2.  Literature review, or understanding the topic, or any other progress you have made.

## 3.  Your plan for the remainder of the semester.

## 4.  Any components/views/structures/elements of the 4+1 Views required for the final project.

## 5. A demo of your working project:
a) From terminal be able to run your scope / function generator.  
b) Show two different UIs.
c) Scroll the data on the screen, showing that if you change the input the display changes.
d) Allow for scaling and offsetting in the process.

Be prepared to do a code review and to discuss how your code maps to the UML diagrams. 

## Things I look for:
a) One class one file. 
b) Name of the class and its meaning.  Use name space in the name of all your classes and files (i.e. scpMyClass -- feel free to use another namespace that is not scp).
c) Responsibility of each class should be clearly listed.
d) I am looking for at least one of each 4+1 views:
    -- requirements, use cases, activity diagrams, class diagrams, object diagrams, sequence diagrams,
    -- state diagrams (optional),
    -- deployment, package.

f) Be ready to put breakpoints and to step through your code!

g) In class presentation.  Submit the power point presentation and the zipped version of your code together with the command for compiling your code (I will compile it).
